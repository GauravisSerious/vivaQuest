✅ Unit 1 — Generative & Probabilistic Models
10 Marks Explain Generative Models and their importance in AI.
+

Definition: Generative Models learn the underlying probability distribution P(X) of training data and can produce brand-new, realistic data samples that statistically resemble the original training distribution.

Importance in AI:

  • Data Augmentation: Generate synthetic labeled samples to expand small datasets — critical in medical imaging where real data is scarce.
  • Image / Video Synthesis: Create photorealistic content — faces, art, game assets, deepfakes.
  • Drug Discovery: Generate novel molecular structures with desired chemical properties, cutting pharma R&D time dramatically.
  • Anomaly Detection: Learn the "normal" data distribution; anything with low likelihood is flagged as an anomaly.
  • Representation Learning: Learn meaningful compressed representations of data (VAE latent space).
  • Density Estimation: Understanding data distribution is fundamental to probabilistic reasoning and Bayesian inference.

⭐ Explain Gaussian Mixture Model (GMM) with working.
+

GMM assumes that all data points are generated from a mixture of K Gaussian (Normal) distributions, each characterized by its own mean (μ), covariance (σ²), and mixing weight (π).

P(x) = Σ πₖ · N(x | μₖ, σₖ²) for k = 1 to K

How GMM Works — EM Algorithm:

  1. Initialize: Randomly assign K Gaussian parameters (μ, σ, π weights).
  2. E-Step (Expectation): For each data point, calculate the probability (responsibility) that it belongs to each Gaussian cluster. This is a soft probabilistic assignment.
  3. M-Step (Maximization): Update μ, σ, and π to maximize the total log-likelihood given the current soft assignments.
  4. Repeat E and M steps until convergence (log-likelihood stops improving).

GMM vs K-Means:

FeatureK-MeansGMM

⭐ Explain Hidden Markov Model (HMM) with example.
+

HMM is a sequential probabilistic model where the system is assumed to be a Markov process with hidden (unobservable) states that produce observable emissions/outputs.

Key Components:

  • Hidden States (S): Unobservable states. E.g., actual Weather = {Sunny, Rainy}.
  • Observations (O): Observable outputs. E.g., Person's activity = {Umbrella, No Umbrella}.
  • Transition Probability (A): P(next state | current state). E.g., P(Rainy | Sunny) = 0.3.
  • Emission Probability (B): P(observation | hidden state). E.g., P(Umbrella | Rainy) = 0.9.
  • Initial State Distribution (π): Probability of starting in each state.

Markov Property:

"The future state depends only on the current state, not the full history." This memoryless property makes HMMs computationally tractable.

HMM vs Markov Chain:

In a Markov Chain, states are directly observable. In HMM, states are hidden — you can only see the emissions they generate, not the states themselves.

Applications:

  • Speech Recognition (phoneme/word identification)
  • Part-of-Speech Tagging in NLP
  • Gesture Recognition
  • Bioinformatics (DNA gene sequence analysis)
⭐ Explain Bayesian Networks and their applications.
+

Bayesian Network is a Probabilistic Graphical Model represented as a Directed Acyclic Graph (DAG) where:

  • Nodes = Random variables (e.g., Disease, Symptom, TestResult).
  • Directed Edges = Conditional dependencies between variables.
  • Each node stores a Conditional Probability Table (CPT) — P(node | parent nodes).

Conditional Probability:

P(A|B) = Probability of A occurring given that B has already happened. Bayesian Networks use Bayes' Theorem: P(A|B) = P(B|A) × P(A) / P(B)

Bayesian Network vs HMM:

HMM is a sequential temporal model (time series). Bayesian Network is more general — models complex non-sequential causal dependency webs between multiple variables.

Applications:

  • Medical Diagnosis: P(Disease | Symptoms, Tests)
  • Spam Filtering: P(Spam | Word frequencies)
  • Fault Diagnosis in industrial systems
  • Risk Assessment in finance & insurance
What are Probabilistic Graphical Models (PGMs)?
+

Probabilistic Graphical Models (PGMs) are a powerful framework that uses graph structures to encode complex joint probability distributions over many random variables. They represent the conditional independence structure among variables compactly.

Two Main Families:

  • Directed Graphical Models (Bayesian Networks): Use directed edges (arrows) to represent causal relationships. Each node is conditionally independent of its non-descendants given its parents.
  • Undirected Graphical Models (Markov Random Fields): Use undirected edges to represent symmetric relationships.

Key Advantage:

PGMs decompose a huge joint distribution P(X₁, X₂, ..., Xₙ) into smaller, manageable local conditional distributions — making inference and learning computationally feasible even for hundreds of variables.

Explain Markov Random Fields (MRF).
+

Markov Random Field (MRF), also called an Undirected Graphical Model, represents the joint probability distribution of variables using an undirected graph. Variables connected by edges are directly dependent; disconnected variables are conditionally independent given their neighbors.

Key Concept — Clique Potentials:

Instead of conditional probability tables, MRFs use potential functions (ψ) defined over cliques (fully connected subgraphs). The joint distribution is proportional to the product of all clique potentials.

P(X) = (1/Z) × ∏ ψ_c(X_c) — Z is the normalizing partition function

MRF vs Bayesian Network:

FeatureBayesian NetworkMRF
Graph TypeDirected Acyclic (DAG)Undirected
RelationshipsCausal (parent → child)Symmetric associations
ParametersConditional Probability TablesPotential / Factor functions

Applications:

Image segmentation (pixels form an MRF where each pixel depends on its neighbors), texture analysis, stereo vision, spatial data modeling.

✅ Unit 2 — GAN (Most Important Unit 🔥)
⭐ Explain GAN Architecture — Generator vs Discriminator.
+

GAN (Generative Adversarial Network) consists of two neural networks trained simultaneously in adversarial competition:

  • Generator G: Takes a random noise vector z as input and maps it to a synthetic data sample (e.g., a fake image). Its goal: fool D into classifying its outputs as real.
  • Discriminator D: Takes either real training data or G's fake output and outputs a probability score (0 = Fake, 1 = Real). Its goal: correctly identify real vs fake.
  • Adversarial Game: As D improves at detecting fakes, G is forced to improve its generation quality. They push each other until G's outputs are indistinguishable from real data.
⭐ Working of GAN — Loss Function & Convergence.
+

GAN training uses a Minimax (Zero-Sum) Loss Function:

min_G max_D V(D,G) = E[log D(x)] + E[log(1 − D(G(z)))]
  • D maximizes: log D(x) → correctly classify real samples as Real (D(x) → 1).
  • D maximizes: log(1 − D(G(z))) → correctly classify fake samples as Fake (D(G(z)) → 0).
  • G minimizes: log(1 − D(G(z))) i.e., maximize D(G(z)) → make D believe its fakes are real.

Training Steps (per iteration):

  1. Sample real batch x from training data.
  2. Sample noise z and generate fake batch G(z).
  3. Update D weights to maximize its discrimination ability.
  4. Update G weights to minimize D's ability to detect its fakes.

Convergence (Nash Equilibrium):

Ideal GAN converges when D(G(z)) = 0.5 for all inputs — D can no longer distinguish real from fake, equivalent to random guessing.

⭐ Challenges in GAN Training — Mode Collapse & Instability.
+
  • Mode Collapse: G finds a narrow set of outputs that reliably fool D and keeps producing only those — completely losing output diversity. E.g., G trained on MNIST only generates "8"s. G has collapsed to a single mode of the data distribution.
  • Training Instability: D and G must stay roughly balanced. If D becomes too strong too fast, G's gradient signal vanishes (nothing to learn from). If G dominates, D cannot provide useful corrective loss.
  • Vanishing Gradient: When D confidently rejects all fake samples (D(G(z)) ≈ 0), log(1-D(G(z))) saturates — gradient reaching G becomes nearly zero.
  • Non-convergence: Training oscillates without ever reaching Nash Equilibrium — G and D keep "chasing" each other.

Solutions:

  • Wasserstein GAN (WGAN): Uses Earth Mover Distance instead of JS Divergence — provides stable, non-vanishing gradients.
  • Mini-batch Discrimination: D evaluates diversity across a mini-batch — penalizes mode collapse.
  • Label Smoothing: Use 0.9 instead of 1.0 for real labels — prevents D from becoming overconfident.
  • Feature Matching: Train G to match statistics of intermediate D features, not just fool the final output.
⭐ Explain DCGAN Architecture and Improvements over vanilla GAN.
+

DCGAN (Deep Convolutional GAN) replaces all fully-connected layers with convolutional layers, making it dramatically more powerful for image generation.

Key Architectural Changes:

  • Convolutional Discriminator: Strided Conv layers downsample the image — the network learns its own spatial downsampling (no max pooling).
  • Transposed Convolution Generator: Fractionally-strided Conv (Deconv) layers progressively upsample noise into a full-resolution image.
  • Batch Normalization: Applied after every layer except D's input and G's output — prevents internal covariate shift, dramatically stabilizes training.
  • Activation Functions: Generator: ReLU (all layers) + Tanh (output). Discriminator: LeakyReLU everywhere (slope=0.2) — avoids dead neurons.
  • No Pooling Layers: Learnable strided convolutions replace all pooling — the network learns its own spatial transformations.

Why use CNN in DCGAN?

Convolutional layers capture spatial hierarchies in images (edges → textures → shapes → objects), making them far superior to dense layers for any visual/image task.

⭐ Explain Conditional GAN (CGAN) — architecture & working.
+

In vanilla GAN, you have no control over what the Generator creates. CGAN (Conditional GAN) solves this by feeding a condition label y (class label, text embedding, or image) as additional input to both G and D.

G input: [z, y] → G(z|y) — Generate sample conditioned on y D input: [x, y] → D(x|y) — Discriminate given condition

Key Difference from Vanilla GAN:

You can explicitly say: "Generate a digit 7" or "Generate a cat." The condition y guides the generation process instead of leaving it to random noise alone.

Applications:

  • Text-to-Image generation (condition = text description)
  • Image-to-Image translation (condition = input image)
  • Super-resolution (condition = low-res image)
  • Facial attribute editing (condition = target attribute like "add glasses")
Explain CycleGAN and its applications.
+

CycleGAN performs unpaired image-to-image translation — converting images from one visual domain to another without requiring matched training pairs.

How it works — Cycle Consistency:

CycleGAN trains two generator pairs: G (X→Y) and F (Y→X). The key innovation is the Cycle Consistency Loss:

X → G(X) → F(G(X)) ≈ X (must reconstruct original) Y → F(Y) → G(F(Y)) ≈ Y (must reconstruct original)

This bidirectional constraint forces the generators to learn meaningful, content-preserving translations even without paired examples.

Applications:

  • Horse ↔ Zebra domain translation
  • Summer ↔ Winter photo conversion
  • Photo → Monet/Van Gogh painting style
  • Medical: MRI ↔ CT scan synthesis (generating one modality from another)
  • Day → Night image conversion for autonomous driving datasets

  • Medical Data Augmentation: Generating synthetic labeled MRI/CT scans to expand rare-disease training datasets without extra patient scans or ethical issues.

⭐ Explain VAE Architecture and Working.
+

  • Encoder: Maps input X to parameters of a probability distribution — outputs mean vector μ and log-variance vector σ (not a fixed point like standard AE).
  • Sampling (Reparameterization Trick): Sample z = μ + σ·ε where ε ~ N(0,1). This trick allows backpropagation through the stochastic sampling step.
  • Decoder: Maps latent vector z back to reconstruct the original input X̂.

Key Idea:

Instead of encoding to a fixed point in latent space, VAE encodes to a probability distribution. This makes the latent space continuous and well-organized, enabling true generative capability by sampling any z from the prior N(0,1).

⭐ Explain the Latent Space in VAE.
+

The Latent Space is a compressed, lower-dimensional mathematical space that captures the essential features and structure of the training data.

  • In a standard Autoencoder: distinct data points map to isolated, scattered points in latent space — space between them is meaningless.
  • In VAE: the KL Divergence loss forces the latent space to follow a standard Gaussian N(0,1) — making it continuous and structured.
  • Interpolation: Smoothly interpolating between two latent points produces semantically meaningful intermediate images. E.g., interpolating between a "smiling face" and "frowning face" point gives a progressively changing expression.
  • Generation: Sample any random z ~ N(0,1) and decode it → always produces a valid, realistic output (e.g., a valid handwritten digit).

Disentangled Latent Space:

Ideally, individual latent dimensions control specific meaningful attributes (e.g., dimension 3 = face rotation, dimension 7 = smile intensity). β-VAE enforces stronger disentanglement.

⭐ Explain the VAE Loss Function — Reconstruction + KL Divergence.
+

VAE Total Loss has two components:

L = Reconstruction Loss + KL Divergence Loss
  • Reconstruction Loss: Measures how accurately the Decoder reproduced the original input. Uses MSE (for continuous data) or Binary Cross-Entropy (for image pixels). Pushes the network to produce faithful reconstructions.
  • KL Divergence Loss: Measures how much the learned latent distribution q(z|x) = N(μ,σ²) diverges from the prior standard normal N(0,1).
KL[q(z|x) || p(z)] = -½ × Σ(1 + log σ² - μ² - σ²)

What each term does:

  • Reconstruction Loss → Encourages accurate data reproduction (decoder quality).
  • KL Loss → Regularizes the latent space to be well-organized and Gaussian-shaped so that any random sample can be decoded into valid data.
  • These two terms create a trade-off: too much KL penalty makes reconstruction blurry; too little makes the latent space fragmented.

Types of Autoencoders — Sparse, Denoising, Contractive.
+
  • Sparse Autoencoder: Adds an L1 sparsity penalty to the hidden layer activations. Forces most neurons to remain zero (inactive) for any given input. This encourages the network to learn a highly compact set of truly important features.
  • Denoising Autoencoder (DAE): The input is artificially corrupted with random noise (Gaussian noise, masking pixels). The network must reconstruct the clean original from the corrupted version. This forces learning of robust, noise-invariant feature representations. Application: image denoising, robustness improvement.
  • Contractive Autoencoder (CAE): Adds a Frobenius norm penalty on the Jacobian of the encoder's activations w.r.t. the input. This penalizes large activation changes in response to small input perturbations — making the encoder robust to tiny input variations.
✅ Unit 4 — Transfer Learning
⭐ What is Transfer Learning? Explain with example.
+

Transfer Learning is the technique of taking a model pre-trained on a massive source dataset and reusing its learned weights as the starting point for a new but related target task — instead of training from random initialization.

Why it works:

Low-level neural network features (edges, textures, colors) are universally useful across visual tasks. A model trained on ImageNet has already learned these fundamental representations — new tasks need only learn high-level task-specific layers.

Example:

  • Take VGG16 pre-trained on ImageNet (1.2M images, 1000 classes).
  • Remove the final 1000-class softmax layer.
  • Add a new Dense layer with 2 output neurons for your task (e.g., X-ray — Pneumonia vs Normal).
  • Fine-tune on just 500 chest X-ray images → Achieves ~90%+ accuracy.

Benefits:

  • Works with very limited labeled data in the target domain.
  • Dramatically reduces training time (hours vs weeks).
  • Significantly lower computational cost.

Best Practice:

Often done in stages: first freeze all layers and train only the head (Feature Extraction). Then unfreeze top conv blocks and fine-tune with a very low learning rate (1e-5 to 1e-4) to gently adapt the model.

Applications of Transfer Learning & Recent Advancements (Self-Supervised, Meta-Learning).
+

Applications:

  • Computer Vision: Medical image diagnosis (X-ray classification, tumor detection) using ResNet/VGG pre-trained on ImageNet.
  • NLP: BERT fine-tuned for sentiment analysis, question answering, named entity recognition.
  • Speech: Wav2Vec pre-trained on unlabeled speech, fine-tuned for low-resource language transcription.
  • Robotics: Pre-trained vision models adapted for robotic manipulation tasks.

Recent Advancements:

  • Self-Supervised Learning: Pre-train on unlabeled data by creating auxiliary tasks (e.g., predict masked image patches — MAE; predict next token — GPT). Eliminates need for expensive labeled pre-training data. Examples: DINO, SimCLR, MAE.
  • Meta-Learning ("Learning to Learn"): Train a model explicitly to rapidly adapt to brand-new tasks with only a few examples (few-shot learning). MAML (Model-Agnostic Meta-Learning) finds an initialization point from which fine-tuning on any new task converges in very few gradient steps.

How Bagging Reduces Overfitting:

By training N models on different random bootstrap subsets and averaging predictions, individual model variances cancel each other out — yielding a stable, generalized ensemble that doesn't memorize noise.

⭐ Explain AdaBoost Algorithm step by step.
+

AdaBoost (Adaptive Boosting) builds a strong classifier by sequentially combining many weak classifiers (typically single-level Decision Stumps):

  1. Initialize: Assign equal weight wᵢ = 1/N to every training sample.
  2. Train Weak Learner: Fit a simple classifier (decision stump) on the weighted dataset.
  3. Compute Error: ε = Σ wᵢ × I(yᵢ ≠ ŷᵢ) — weighted misclassification rate.
  4. Compute Learner Weight: α = ½ × ln((1−ε)/ε). High accuracy → high α (more say in final vote).
  5. Update Sample Weights: Increase weights of all misclassified samples → next learner focuses harder on hard examples. Decrease weights of correctly classified samples.
  6. Normalize weights so they sum to 1. Repeat from step 2 for T rounds.
  7. Final Prediction: Weighted majority vote: H(x) = sign(Σ αₜ × hₜ(x)).

Limitation: Very sensitive to noisy data and outliers — continuously upweights genuinely mislabeled examples, degrading performance.

⭐ Explain XGBoost and its advantages.
+

XGBoost (Extreme Gradient Boosting) is a highly engineered, scalable implementation of gradient-boosted decision trees. It consistently dominates ML competitions (Kaggle).

Key Advantages over standard Gradient Boosting:

  • Built-in L1 & L2 Regularization: Directly penalizes model complexity in the objective function — prevents overfitting that vanilla GBM lacks natively.
  • Parallel Tree Building: Builds tree nodes level-by-level using multi-threaded parallel processing — significantly faster training.
  • Approximate Split Finding: Uses histogram-based quantile sketching to find optimal splits without scanning all data — scalable to huge datasets.
  • Handles Missing Data: Automatically learns the optimal default direction for missing value splits during training — no manual imputation needed.
  • Efficient Tree Pruning: Grows trees to max_depth then prunes backwards using a gain threshold (max_delta_step) — more conservative than greedy early stopping.
  • Sparsity Awareness: Optimized for sparse input matrices — common with one-hot encoded categorical features.

XGBoost Objective Function:

L = Σ l(yᵢ, ŷᵢ) + Σ Ω(fₖ) where Ω(f) = γT + ½λ||w||² (tree complexity penalty)

🎯 Practical Viva — Quick Answer Format
Pract: HMM What is HMM? Where is it used? Markov Property? HMM vs Markov Chain? Transition vs Emission Probability?
+
  • What is HMM? A sequential probabilistic model with hidden states that generate observable emissions. You can observe the outputs but not the states directly.
  • Where used? Speech recognition (phoneme modeling), POS tagging in NLP, gesture recognition, DNA sequence analysis, financial market modeling.
  • Markov Property: "The next state depends ONLY on the current state — not the full history." This memoryless assumption simplifies computation enormously.
  • HMM vs Markov Chain: Markov Chain = states are directly observable. HMM = states are hidden; you observe only the emissions they produce.
  • Transition Probability: P(state at t+1 | state at t). How likely to move from one hidden state to another. E.g., P(Rainy tomorrow | Sunny today) = 0.3.
  • Emission Probability: P(observation | hidden state). What observable output a state tends to produce. E.g., P(carry umbrella | Rainy state) = 0.9.
Pract: GMM What is GMM? GMM vs K-Means? EM Algorithm? Why is GMM Probabilistic?
+
  • What is GMM? A probabilistic clustering model that assumes data is generated from a mixture of K Gaussian distributions, each with its own μ, σ, and weight π.
  • GMM vs K-Means: K-Means makes hard cluster assignments; GMM gives soft probabilistic memberships (point can be 70% Cluster A, 30% Cluster B). GMM handles elliptical clusters; K-Means only spherical.
  • EM Algorithm: E-Step: Compute probability each point belongs to each cluster (soft assignment). M-Step: Update μ, σ, π to maximize log-likelihood. Repeat until convergence.
  • Why GMM is Probabilistic: Unlike K-Means' hard binary assignment, GMM outputs a probability distribution over cluster memberships for every data point — capturing uncertainty in cluster assignment.
Pract: Bayesian Net What is Bayesian Network? Conditional Probability? BN vs HMM?
+
  • What is Bayesian Network? A Directed Acyclic Graph where nodes = random variables, edges = conditional dependencies. Each node stores P(node | parents) in a Conditional Probability Table (CPT).
  • Conditional Probability: P(A|B) = probability of A given that B has occurred. Computed via Bayes' Theorem: P(A|B) = P(B|A)×P(A) / P(B).
  • BN vs HMM: HMM is a specialized sequential temporal model (for time-series data with hidden states). Bayesian Network is more general — models arbitrary complex, non-sequential causal dependencies between multiple variables.
Pract: GAN/DCGAN/CGAN What is GAN? Generator/Discriminator roles? DCGAN? CGAN? Mode Collapse? Unstable Training?
+
  • What is GAN? Two competing neural networks — Generator creates fake data from noise; Discriminator tries to distinguish real from fake. Their adversarial competition improves both models.
  • Generator: Takes random noise z → produces synthetic data. Goal: fool D so D(G(z)) → 1.
  • Discriminator: Takes real or fake data → outputs probability of being real. Goal: D(real) → 1, D(fake) → 0.
  • DCGAN: Uses Convolutional layers instead of Dense layers. Adds BatchNorm. Uses LeakyReLU in D and ReLU+Tanh in G. No pooling layers. Far better for image tasks — CNNs capture spatial hierarchies.
  • Conditional GAN: Adds condition label y to both G and D inputs — allows controlled, targeted generation (e.g., "generate only digit 3").
  • Mode Collapse: G fixates on generating only a narrow set of outputs that reliably fool D — losing diversity. G has not learned the full data distribution.
  • Unstable Training: G and D must remain balanced. If D dominates, G gets vanishing gradients. If G dominates, D loses ability to give useful loss signals. Result: training oscillates or diverges.
Pract: VAE What is VAE? Latent Space? Why Probabilistic? AE vs VAE?
+
  • What is VAE? A generative model combining an Encoder (maps input to a probability distribution μ, σ) and Decoder (reconstructs input from a sampled latent vector z). Trained with Reconstruction + KL Divergence loss.
  • Latent Space: Compressed, lower-dimensional representation space. VAE forces this to follow N(0,1) — making it continuous and smooth. Any point in latent space decodes into a valid output.
  • Why VAE is Probabilistic: Encoder produces a distribution (μ, σ) rather than a fixed latent point. At inference, you sample z from this distribution — enabling stochastic, diverse generation.
  • AE vs VAE: AE encodes to a fixed point, cannot generate new data, has fragmented latent space. VAE encodes to a distribution, can generate new data, has smooth continuous latent space.
Pract: Transfer What is a pre-trained model? Why use Transfer Learning? Feature Extraction vs Fine-Tuning?
+
  • Pre-trained Model: A model already trained on a large source dataset (e.g., VGG16 on ImageNet 1.2M images). Its weights encode general knowledge — low-level visual features (edges, textures) and high-level semantic features (objects, scenes).
  • Why use Transfer Learning? (1) Limited labeled data for target task. (2) Massive compute savings — no training from scratch. (3) Pre-trained features generalize well. Training ResNet from scratch takes weeks on 8 GPUs; fine-tuning takes hours.
  • Feature Extraction: Freeze ALL pre-trained weights. Only train newly added head layers. Best when source and target tasks are very similar and target dataset is tiny.
  • Fine-Tuning: Unfreeze some or all pre-trained layers and retrain with a very small learning rate alongside the new head. Best when target task differs somewhat from source, and you have a moderate-sized dataset.
Pract: Ensemble Bagging? Boosting? Difference? Overfitting & how Bagging reduces it?
+
  • Bagging: Train multiple independent models on different random bootstrap samples of training data simultaneously. Combine predictions via majority vote (classification) or averaging (regression). Example: Random Forest.
  • Boosting: Train models sequentially. Each new model focuses specifically on the errors of the previous one (misclassified samples get higher weight). Combine via weighted vote. Example: AdaBoost, XGBoost.
  • Key Difference: Bagging = Parallel, reduces Variance (overfitting). Boosting = Sequential, reduces Bias (underfitting).
  • Overfitting: Model memorizes training noise → very high training accuracy, poor test accuracy. Detected by large gap between training and validation performance.
  • How Bagging Reduces Overfitting: By training on different random subsets and averaging predictions, individual model variance (sensitivity to training noise) cancels out across the ensemble. The averaged prediction is much smoother and more generalized than any single model.
