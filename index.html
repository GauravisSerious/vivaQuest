🔹Section 1: Project Basics

What is the full form of AAKAR?
Autism Assistive Kit for Awareness & Recognition. "Aakar" (आकार) means "shape/form" in Hindi.
What problem does your project solve?
Children with Autism Spectrum Disorder (ASD) struggle to recognize and express emotions. Aakar uses AI-powered emotion detection combined with therapeutic games to help them learn emotions interactively.
Who are the target users?
Five roles: Child (primary learner), Teacher (classroom monitoring), Therapist (therapy management), Parent (progress tracking), Admin (system management).
What is the objective of your project?
To develop a cross-platform assistive application that detects facial emotions using AI (on-device + cloud), teaches emotions through 15+ gamified activities, and enables collaborative progress tracking across teachers, therapists, and parents.
Is this mobile, web, or desktop?
All three. Built with Flutter — runs on Android, iOS, Windows, macOS, Linux, and Web from a single codebase.
What makes your project unique?
  • Dual AI approach — on-device TFLite (offline) + cloud DeepFace (high accuracy)
  • Live real-time detection via WebView + face-api.js
  • Multi-stakeholder dashboards — 4 dedicated dashboards for different roles
  • Privacy-first — all data stored locally, no cloud dependency
  • Bilingual — English + Hindi (200+ strings each)

Why Flutter?
Single codebase for 6 platforms, hot reload for rapid development, rich widget library for child-friendly UI, and strong community support.
Why FastAPI and not Django/Flask?
FastAPI is async by default, has automatic Swagger docs, and is 3-4x faster than Flask. Built-in request validation via Pydantic.
Why SQLite instead of Firebase?
No internet required (works offline), privacy (children's data stays on-device), zero configuration (no server setup needed).
Why Provider and not BLoC or GetX?
Officially recommended by the Flutter team, minimal boilerplate, and our project's 8 providers are manageable without BLoC's complexity.

🔹Section 3: AI / Machine Learning

How does emotion detection work?

What is DeepFace?
An open-source Python library by Facebook Research that wraps multiple deep learning models for facial analysis — including emotion, age, gender, and race prediction.
What model/dataset is used?
FER2013 (Facial Expression Recognition 2013) — 35,887 grayscale 48×48 images across 7 emotions: Happy, Sad, Angry, Fear, Surprise, Neutral, Disgust.
What are the 7 emotions detected?
😊 Happy, 😢 Sad, 😠 Angry, 😨 Fear, 😲 Surprise, 😐 Neutral, 🤢 Disgust
What is TFLite?
TensorFlow Lite — a lightweight version of TensorFlow optimized for mobile and edge devices. Runs ML models directly on the phone without a server.
How does live detection work without internet?
Flutter reads face-api.js + model weights from local assets → encodes as base64 → injects into WebView. A fetch polyfill intercepts network requests and serves models from memory. Zero CDN dependency.
What is Haar Cascade?
A classical machine learning algorithm for object detection. We use OpenCV's pre-trained classifier to detect and crop faces from images before passing them to DeepFace.
What preprocessing do you apply?
  • Image resized to 512×512 px
  • JPEG compressed to ≤500KB
  • Face detected and cropped to 224×224 px using Haar Cascade
  • Grayscale conversion for the FER2013 model
What is the model accuracy?
FER2013 baseline: ~65-68% (well-known limitation). DeepFace with MTCNN: ~82-85%. We apply post-processing corrections (neutral penalty ×0.3, fear/surprise heuristic) to boost real-world performance.
What corrections do you apply?
  • Neutral penalty (×0.3) — FER2013 is biased toward "Neutral"
  • Fear ↔ Surprise heuristic — model confuses these; we apply probabilistic correction
  • Linear normalization — ensures probabilities sum to 1.0

🔹Section 4: Architecture & Design

Explain the system architecture.
Three-tier architecture:
  • Presentation Layer — Flutter UI (pages, widgets)
  • Business Logic Layer — Providers (state) + Services (API calls, ML)
  • Data Layer — SQLite + Hive + SharedPreferences
Backend is optional — app works fully offline via TFLite.
How does frontend communicate with backend?
Via REST API over HTTP. POST /analyze-emotion sends image (multipart), receives JSON with emotion probabilities. Uses adb reverse for USB or WiFi IP for wireless.
What design pattern do you use?
MVVM (Model-View-ViewModel) using Provider: Model = lib/models/, View = lib/pages/, ViewModel = lib/providers/.
How is state managed?
8 ChangeNotifier Providers: AuthProvider (login), SettingsProvider (theme/API), LanguageProvider (EN↔HI), EmotionHistoryProvider (scans), GameProvider (scores), RoleProvider (role), ChatbotProvider (chat), OnboardingProvider (first-launch).
How many screens does the app have?
50+ screens — 4 dashboards, 15+ games, detection screens, settings, and sub-pages for each role.

How are passwords stored?
Using bcrypt hashing with salt. Passwords are never stored in plaintext. On login, the entered password is hashed and compared against the stored hash.

🔹Section 6: Key Features

How does Live Emotion Detection work?
  • Flutter opens a WebView with an HTML page
  • Injects face-api.js + model weights as base64 from local assets
  • A fetch polyfill intercepts model requests → serves from memory
  • Camera opens → face detection runs every 400ms
  • Results sent to Flutter via JavaScript channel (EmotionChannel.postMessage)

How does bilingual work?
All strings stored in localization/english.dart and hindi.dart (200+ each). A translate() function returns the correct string based on LanguageProvider's current locale.

🔹Section 7: Security & Privacy

How do you handle children's data privacy?
All data stored locally on device (SQLite). Backend is optional and only processes face images temporarily (not stored). Camera permissions requested at runtime with graceful denial handling.
Is the API secure?
Development uses HTTP locally. For production: HTTPS with SSL/TLS, API key authentication, restricted CORS, and rate limiting.
